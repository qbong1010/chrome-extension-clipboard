# Chrome Web Store 등록 준비 가이드

이 문서는 Paste Right 확장 프로그램을 Chrome Web Store에 등록하기 위해 개발자 대시보드에서 작성해야 할 필수 항목들을 안내합니다.

## 1. 스토어 기본 정보 (스토어 등록정보 탭)
- **이름**: Paste Right
- **요약**: (최대 132자 이내) 우클릭 한번으로 자주 쓰는 메시지를 즉시 입력합니다.
- **상세 설명**:
  - 확장 프로그램의 주요 기능(템플릿 관리, 우클릭 메뉴 삽입 등)과 사용 방법을 사용자에게 설명하십시오.
- **카테고리**: 생산성 도구 (또는 적합한 서브카테고리 설정)
- **언어**: 한국어

## 2. 그래픽 자산 (필수)
- **스토어 아이콘**: `128 x 128 px`, PNG 형식
- **스크린샷**: 최소 1장 ~ 최대 5장, `1280 x 800 px` 또는 `640 x 400 px` 크기의 PNG/JPEG
  - ※ *추천*: 사이드패널에서 템플릿을 고르는 화면, 우클릭 메뉴에서 템플릿을 선택하는 화면의 스크린샷 2장을 제출하세요.
- **프로모션 타일** (선택): 440 x 280 px

## 3. 권한 설명 (개인정보 보호 탭)
심사 시 각 권한을 사용하는 이유에 대해 명확한 소명이 필요합니다:
- **`activeTab`**: 현재 활성화된 탭에서 클립보드 데이터를 텍스트 입력창 등에 붙여넣기 위해 일시적인 권한이 필요합니다.
- **`contextMenus`**: 사용자가 마우스 우클릭을 통해 쉽고 빠르게 저장된 템플릿을 선택하고 입력할 수 있도록 메뉴 아이템을 구성하기 위해 사용합니다.
- **`scripting`**: 선택된 템플릿 코드를 `activeTab`에 JavaScript를 통해 삽입(`executeScript`)하기 위해 사용합니다.
- **`storage`**: 사용자가 저장한 템플릿 목록 및 확장 프로그램 설정 옵션을 로컬에 저장하기 위해 사용합니다.

## 4. 데이터 취급(개인정보 보호 탭)
- **개인정보처리방침 (Privacy Policy)**: `docs/PRIVACY_POLICY.md` 파일의 내용을 바탕으로 생성된 페이지의 URL 혹은 Google Docs, Github Pages 등의 URL 링크 제공
- 데이터 수집 등 사용자 정보 관련 체크박스에서는 **데이터를 전송/수집하지 않음**으로 명확히 체크하세요.

## 5. 업로드 파일
- `npm run package:extension` 명령을 통해 생성된 `dist/paste-right-v1.5.zip` 파일을 업로드하세요. 완성된 패키지 안에는 필수적인 `icons` 디렉터리와 필요한 manifest.json 설정이 포함되어 있습니다.
